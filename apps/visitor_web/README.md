# VisitFlow Visitor Web

This workspace is reserved for the lightweight visitor-facing portal used for:

- entrance QR registration;
- invitation confirmation;
- privacy consent;
- digital visitor passes;
- visit cancellation;
- visitor check-out.

The Next.js application is intentionally deferred to a separate foundation pull request after the Flutter staff shell passes CI. Keeping these changes separate makes failures easier to diagnose and prevents unrelated framework setup from being bundled into one review.

Visitors will not be required to install the Flutter staff application.
